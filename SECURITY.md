# Security Policy

If you discover a security vulnerability in this project, please report it privately.

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` branch | Yes |
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security problems.**

Report a vulnerability through one of these private channels:

- **GitHub Security Advisories:** <https://github.com/mThanuj/dsa/security/advisories/new>
- **Email:** <thanujmullaguri1825@gmail.com>

Please include:

- A description of the issue and its impact.
- Steps to reproduce, or a minimal proof of concept.
- The affected version or commit.

We will acknowledge your report within a few days and work with you on a fix and coordinated disclosure.

## Scope

This is a small, pure-source library of algorithm implementations. There is no network surface, no authentication, and no user data, so the attack surface is limited. Reports of crashes or incorrect behavior caused by inputs that are out of contract should be filed as regular [bug reports](CONTRIBUTING.md) instead.
